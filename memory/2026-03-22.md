## OpenClaw MCP / Chase API auth + syntax
- **2026-03-22:** Chase MCP endpoint at `https://life-tracker-production-17cb.up.railway.app/api/mcp` requires `Accept: application/json,text/event-stream` for MCP requests; using only JSON accept headers can fail even when auth syntax looks otherwise correct.
- **Auth status lesson:** unauthorized responses during testing were resolved after updating the API key stored at `/data/workspace/.chase_api_key`; do **not** store raw keys in memory files.
- **Verified flow:** MCP `tools/list` succeeded after key refresh and corrected headers; Chase task `"Bug: OpenClaw MCP calls fail — wrong JSON-RPC method format"` was then marked done successfully via MCP.

## Attachment delivery behavior
- **2026-03-22:** Inbound attachments are only directly readable when OpenClaw injects a real saved path (for example `/data/.openclaw/media/inbound/...`).
- A message that only shows `<media:document>` / reply context may **not** expose the actual file path to the session, even if the user clearly replied to the file.
- Later “Queued messages while agent was busy” deliveries can include `[media attached: <full path>]`, which makes the file readable immediately.
- Conclusion: failures to read the earlier PPTX were due to attachment handoff / path injection differences, **not** necessarily file size.
