## Knowledge saved: OpenAI “Harness engineering”
- Source: https://openai.com/index/harness-engineering/
- Core idea: in an agent-first setup, **humans steer, agents execute**; leverage comes from designing the harness (environment, rules, feedback loops), not hand-writing code.
- Practical patterns worth reusing:
  - Keep `AGENTS.md` short (table of contents), move deep guidance into structured docs.
  - Prefer progressive disclosure + enforce doc freshness with linters/CI.
  - Enforce architectural invariants mechanically (boundaries, layering, naming, schema checks).
  - Make runtime observable to agents (UI access, logs, metrics, traces) so they can self-debug.
  - Use iterative agent-review loops and encode human feedback into reusable rules.
  - Run continuous “garbage collection” (small recurring cleanup tasks) to prevent AI-generated drift/slop.

## Why this matters for SpongeBot
- Improve reliability by tightening triggers, constraints, and SOP clarity instead of ad-hoc prompting.
- Prioritize codifying rules into SOP/docs/config so quality compounds over time.
